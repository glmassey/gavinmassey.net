AWS Setup for Terraform

Before running Terraform to create an AWS EC2 instance, you need to set up your AWS account and credentials.

1. Create an AWS Account

  • If you don’t already have an AWS account, sign up at AWS Management Console.

2. Create an IAM User

  • Log into the AWS Management Console.
  • Navigate to the IAM (Identity and Access Management) service.
  • Create a new IAM user with programmatic access. This provides you with an access key ID and secret access key, which are needed to allow Terraform to manage resources on your behalf.
  • Attach policies granting necessary permissions (e.g., AmazonEC2FullAccess).

3. Find the Ubuntu AMI

  • Navigate to the EC2 service in the AWS Management Console.
  • Click on Launch Instance to find the AMI (Amazon Machine Image) ID for the latest Ubuntu LTS version. Make sure you select an AMI in the region you plan to use.
  • Note down the AMI ID as you will need to replace ami-12345678 in main.tf with this ID.

4. Install the AWS CLI

  • Install the AWS Command Line Interface (CLI) as per instructions here: Installing AWS CLI.

5. Configure AWS CLI

  • Run aws configure in your command line.
  • Enter your AWS Access Key ID, Secret Access Key, and default region name.

6. Running Terraform

  • Initialize Terraform in the directory where your main.tf is located by running terraform init.
  • Apply the Terraform configuration with terraform apply.
  • Confirm the action and Terraform will create the EC2 instance.

7. Accessing the Instance

  • Once the instance is created, you can find its public IP address in the AWS EC2 console.
  • To access the instance, you’ll need an SSH key pair. You can create this in the EC2 console under Key Pairs.

8. Clean Up Resources

  • Remember to run terraform destroy when you’re done to avoid ongoing charges for the AWS resources.
