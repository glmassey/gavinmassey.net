YAML Basics

1. Structure:

  • YAML uses indentation (spaces, not tabs) to represent the structure, similar to Python.
  • The number of spaces in the indentation is not fixed, but all elements of the list or map must be aligned.

2. Scalars (Strings, Numbers, Booleans):

  • Strings can be unquoted or enclosed in single (') or double quotes (").
  • Numbers are written as in most programming languages.
  • Booleans are represented with true and false.

3. Lists (Arrays):

  • Items in a list are marked with a hyphen and a space (-).
  • Lists can be ‘in-line’ using square brackets [ ].

Example:

fruits:
  - Apple
  - Banana
  - Cherry
numbers: [1, 2, 3]

4. Maps (Dictionaries or Objects):

  • Maps are collections of key-value pairs.
  • The key-value pairs are separated by a colon and space (:).

Example:

person:
  name: John Doe
  age: 30

5. Nested Structures:

  • You can create nested lists and maps.
  • Indentation is used to represent hierarchy.

Example:

employees:
  - name: John
    job: Developer
  - name: Jane
    job: Designer

6. Comments:

  • Comments begin with a #.

Example:

# This is a comment
name: John Doe

7. Multiline Strings:

  • Use | for a literal block, where newlines are preserved.
  • Use > for a folded block, where newlines become spaces.

Example:

literal: |
  This is a multiline
  string.  
folded: >
  This is a multiline
  string too.

8. Anchors and Aliases:

  • You can define an anchor (&) and use an alias (*) to repeat content.

Example:

defaults: &defaults
  car: Tesla
  color: black

mycar:
  <<: *defaults
  model: Model S

Best Practices

  • Consistent indentation (usually 2 spaces).
  • Use comments for documentation.
  • Keep it simple and readable.

YAML is a flexible and user-friendly format for configuration files and data representation. While it’s simple at first glance, it offers powerful features for more complex structures. Always validate your YAML files, especially when using them in production environments.
