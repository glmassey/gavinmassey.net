Nslookup Quick Reference Guide

Introduction

nslookup is a network administration command-line tool available in most operating systems, including Windows, macOS, and Linux. It is used for querying the Domain Name System (DNS) to obtain domain name or IP address mapping information. This tool is essential for network diagnostics and troubleshooting DNS issues.

Key Features of nslookup

  1. Query DNS Records: nslookup can query specific DNS record types, such as A (address), MX (mail exchange), NS (name server), and TXT (text) records.

  2. Diagnose DNS Servers: It’s useful for testing and troubleshooting DNS server responses to ensure they are correctly resolving domain names.

  3. Reverse DNS Lookups: Besides standard queries (domain to IP), nslookup can perform reverse lookups (IP to domain).

  4. Check Domain Information: Provides detailed information about domains, including the registrar, name server, and other DNS records.

Using nslookup

  1. Standard DNS Query:

    nslookup example.com

    This command queries the DNS records for example.com, returning its IP address.

  2. Query Specific DNS Record Types:

    nslookup -type=mx example.com

    Queries the MX records for example.com, which are used for email.

  3. Reverse DNS Lookup:

    nslookup 93.184.216.34

    Performs a reverse DNS lookup on the IP address to find its associated domain name.

  4. Specify a DNS Server for the Query:

    nslookup example.com 8.8.8.8

    Queries example.com using Googleâ€™s DNS server (8.8.8.8).

  5. Interactive Mode:

    nslookup
> set type=ns
> example.com

    Enters interactive mode, sets the query type to NS (Name Server), and then queries the name servers for example.com.

Conclusion

nslookup is a powerful tool for network administrators and IT professionals for diagnosing and resolving DNS-related issues. Whether it’s verifying the correct resolution of domain names, investigating mail server problems, or gathering information about domains, nslookup provides a straightforward and effective way to interact with the DNS system.
