Value proposition of the AWS Cloud

AWS Cloud offers a broad set of global cloud-based products including compute, storage, databases, analytics, networking, mobile, developer tools, management tools, IoT, security, and enterprise applications. These services help organizations move faster, lower IT costs, and scale applications. Key benefits include agility and innovation, elasticity, a pay-as-you-go pricing model, global reach, and a broad set of tools and services.

AWS Well-Architected Framework

The AWS Well-Architected Framework provides a consistent approach for customers and partners to evaluate architectures and implement scalable designs. It consists of five pillars: Operational Excellence, Security, Reliability, Performance Efficiency, and Cost Optimization. Each pillar focuses on specific aspects of architecture and provides a set of best practices and guidelines.

Cloud adoption strategies

Cloud adoption strategies involve the plans and approaches that organizations take to move their operations to the cloud. Key strategies include re-hosting (lift-and-shift), re-platforming (lift, tinker, and shift), re-purchasing (move to a different product), refactoring/re-architecting (reimagine how the application is architected and developed), and retain (keep applications on-premises) or retire (get rid of outdated applications).

Resources to support the cloud migration journey

Resources for cloud migration include AWS Migration Hub for tracking progress, AWS Application Discovery Service for understanding on-premises applications, AWS Database Migration Service for moving databases, AWS Server Migration Service for moving on-premises servers, and AWS Snow Family for large-scale data transfers. AWS also offers various support plans, training, and professional services to assist in the migration.

Aspects of cloud economics

Cloud economics refers to the financial aspects of cloud computing and includes concepts like cost savings, operational expenses (OpEx) versus capital expenses (CapEx), and the total cost of ownership (TCO). It also involves understanding and managing costs with tools like AWS Cost Explorer and AWS Budgets, and optimizing expenses with services like AWS Trusted Advisor and AWS Compute Optimizer.

Cost savings of moving to the cloud

Moving to the cloud can lead to significant cost savings due to reduced upfront capital expenditure. Instead of investing in data centers and servers before knowing how they will be used, cloud computing allows businesses to pay only for the compute power, storage, and other resources they use. The variable expense model eliminates the need for guessing capacity needs, and economies of scale mean lower pay-as-you-go prices.

AWS shared responsibility model

The AWS shared responsibility model outlines that AWS is responsible for securing the infrastructure that runs AWS services in the AWS Cloud, while AWS customers are responsible for securing their data and applications in the cloud. This includes managing the guest operating system, configuring AWS-provided security group firewalls, and encrypting data.

AWS compliance and governance concepts

AWS provides several compliance and governance tools to ensure that cloud infrastructure meets required standards and regulations. AWS Artifact offers on-demand access to AWS compliance reports. AWS Config helps track and manage configurations. AWS CloudTrail provides governance, compliance, and operational and risk auditing of the AWS account. These tools assist in maintaining compliance with various regulatory standards.

Benefits of cloud security (for example encryption)

Cloud security, such as encryption, offers significant benefits in protecting data. Encryption enhances data privacy and security, making it unreadable without the correct decryption keys, thus protecting it from unauthorized access. AWS provides various encryption options, both in transit and at rest, across its services, ensuring that sensitive data is securely managed.

Where to capture and locate logs that are associated with cloud security

Logs related to cloud security can be captured and located using services like Amazon CloudWatch, which collects and tracks metrics, collects and monitors log files, and sets alarms. AWS CloudTrail further supports security by providing a history of AWS API calls for an account, including API calls made via the AWS Management Console, AWS SDKs, command line tools, and other AWS services. These logs are invaluable for auditing and security monitoring.

Identity and access management (for example, AWS Identity and Access Management [IAM])

AWS Identity and Access Management (IAM) allows you to manage access to AWS services and resources securely. Using IAM, you can create and manage AWS users and groups, and use permissions to allow and deny their access to AWS resources. IAM is integral to AWS security and enables you to control who is authenticated (signed in) and authorized (has permissions) to use resources.

Importance of protecting the AWS root user account

The AWS root user account has full access to all resources in the AWS account. It is critical to protect this account to prevent unauthorized access that could lead to potential security risks or misuse. Best practices include using a strong password, enabling multi-factor authentication (MFA), and limiting the use of the root account, preferring IAM users with necessary permissions for daily tasks.

Principle of least privilege

The principle of least privilege in AWS means giving a user permission only for the resources and actions they need to perform their job and nothing more. This minimizes the risk of an unauthorized access or accidental changes to sensitive resources. Regularly reviewing and revising permissions ensures that users have only the access they require.

AWS IAM Identity Center (AWS Single Sign-On)

AWS IAM Identity Center (formerly AWS Single Sign-On) simplifies the process of managing access to AWS accounts and business applications. It enables users to sign in to a central portal to access all their assigned AWS accounts and applications. This centralized access helps in managing permissions and streamlines the process for users to gain access to the resources they need.

Security capabilities that AWS provides

AWS offers a comprehensive set of security capabilities to increase privacy and control network access. These include network firewalls built into Amazon VPC, encryption in transit with TLS across all services, private or dedicated connections from your office or on-premises environment, customer-controlled encryption in transit and at rest, and identity and access control to secure access to AWS services. AWS infrastructure is designed to keep data safe, with continuous compliance checks, and automated threat detection and mitigation.

Security-related documentation that AWS provides

AWS provides extensive security-related documentation to help users understand and implement best practices for securing their AWS environment. This includes the AWS Security Best Practices guide, AWS Whitepapers on various security topics, AWS compliance resources, and detailed documentation for individual AWS services addressing specific security features and configurations. These resources are essential for understanding AWS security protocols, recommended practices, and for staying updated on the latest security developments.

Different ways of provisioning and operating in the AWS Cloud

Provisioning and operating in the AWS Cloud can be done in various ways, including the AWS Management Console (a web interface), AWS Command Line Interface (CLI), AWS Software Development Kits (SDKs) for different programming languages, and Infrastructure as Code tools like AWS CloudFormation. These methods offer flexibility in managing resources, automating processes, and integrating AWS services into various applications and workflows.

Different ways to access AWS services

AWS services can be accessed in several ways: through the AWS Management Console (a web-based interface), the AWS Command Line Interface (CLI), AWS SDKs (for programmatic access using various programming languages), RESTful APIs (to integrate with AWS services programmatically), and AWS Tools for PowerShell (for those who prefer to work within a PowerShell scripting environment).

Types of cloud deployment models

Cloud deployment models include public cloud (services delivered over the public internet and shared across organizations), private cloud (dedicated to a single organization, either on-premises or hosted by a third-party), hybrid cloud (a combination of public and private clouds, offering a mix of internal and external services), and multicloud (using multiple cloud computing and storage services in a single network architecture).

Connectivity options

Connectivity options to AWS include the public internet, AWS Direct Connect (a cloud service solution that establishes a direct, private network connection from an organization to AWS), VPN connections (secure, private connections over the internet), and AWS Transit Gateway (connecting VPCs and on-premises networks through a central hub). These options provide varying balances of speed, security, and cost, depending on organizational needs.

AWS Regions, Availability Zones, and edge locations

AWS Regions are separate geographic areas that host AWS cloud infrastructure. Each Region consists of multiple, isolated locations known as Availability Zones, which are designed to be interconnected with low-latency network links. Edge locations are sites deployed in major cities and highly populated areas worldwide, used by Amazon CloudFront (AWS’s content delivery network) to cache data and reduce latency for end-user access.

High availability

High availability in AWS refers to a system design that ensures a certain level of operational performance, usually uptime, for a higher than normal period. This is achieved by minimizing points of failure, using multiple Availability Zones within a Region for redundancy, and ensuring automatic failover and load balancing. High availability is crucial for critical business operations and services.

Use of multiple Regions

Using multiple AWS Regions can enhance disaster recovery and business continuity, provide lower latency by serving users from the nearest Region, and allow for localized data storage in compliance with geographic governance and data residency requirements. Multi-region deployment can also offer increased scalability and fault tolerance.

Benefits of edge locations

Edge locations improve content delivery and reduce latency by caching data closer to users. They are particularly beneficial for accelerating static content delivery (like images, videos, stylesheets, and scripts), streaming content, and reducing the load on origin servers. Edge locations can also handle requests for dynamic content that cannot be cached, by routing these requests to the nearest AWS Region.

AWS Wavelength Zones and AWS Local Zones

AWS Wavelength Zones are infrastructure deployments embedded within telecommunications providersâ€™ datacenters at the edge of the 5G network, enabling developers to build applications that require ultra-low latencies over the 5G network. AWS Local Zones, on the other hand, are extensions of AWS Regions placed in specific geographic locations, closer to end-users, to offer lower latency and high-bandwidth connectivity for latency-sensitive applications.

AWS compute services

AWS compute services provide scalable computing capacity in the cloud. Key services include Amazon EC2 (Elastic Compute Cloud) for resizable compute capacity, AWS Lambda for running code without provisioning servers, AWS Elastic Beanstalk for deploying and managing applications, and AWS Fargate for running containers without managing servers or clusters.

AWS database services

AWS database services offer a variety of database solutions to fit different needs. Amazon RDS (Relational Database Service) simplifies setting up, operating, and scaling a relational database. Amazon DynamoDB offers fast and flexible NoSQL database service, while Amazon Aurora provides a high performance managed relational database. There are also specialized services like Amazon Redshift for data warehousing and Amazon Neptune for graph databases.

Database migration

AWS provides services and tools for database migration, notably AWS Database Migration Service (DMS), which helps in migrating databases to AWS quickly and securely. The service supports both homogenous migrations (like MySQL to MySQL) and heterogeneous migrations (like Oracle to Amazon Aurora).

AWS network services

AWS network services ensure secure and efficient networking within the AWS ecosystem. These include Amazon VPC (Virtual Private Cloud) for provisioning a logically isolated section of AWS, AWS Direct Connect for establishing a dedicated network connection to AWS, and Amazon Route 53 for scalable DNS and domain name registration. Other services include AWS Transit Gateway for network scaling and AWS Elastic Load Balancing for distributing incoming traffic across multiple targets.

AWS storage services

AWS offers a range of storage services for various needs. Amazon S3 (Simple Storage Service) is ideal for scalability and data availability, suitable for ‘data lakes’ and backup storage. Amazon EBS (Elastic Block Store) provides block-level storage volumes for persistent data storage with EC2 instances. Amazon Glacier is used for long-term archival storage. AWS also has specialized storage services like AWS Storage Gateway, which integrates on-premises IT environments with cloud storage, and Amazon FSx, providing fully managed third-party file systems.

AWS AI/ML services

AWS AI/ML services facilitate machine learning and artificial intelligence capabilities for a wide range of users, from developers to data scientists. Amazon SageMaker helps in building, training, and deploying machine learning models at scale. Amazon Rekognition offers image and video analysis, while Amazon Polly turns text into lifelike speech. Amazon Lex provides conversational interfaces using voice and text, and Amazon Forecast and Amazon Personalize deliver AI-driven forecasting and recommendations.

AWS analytics services

AWS analytics services provide tools to process, analyze, and visualize big data and analytics workloads. Amazon Redshift offers a fast, scalable data warehouse service. Amazon EMR provides a managed Hadoop framework for processing vast amounts of data. AWS Glue is a serverless data integration service for preparing and combining data for analytics, and Amazon Athena allows querying data in Amazon S3 using SQL. Amazon QuickSight is used for business intelligence and data visualization.

Application integration services of Amazon EventBridge, Amazon Simple Notification Service (Amazon SNS), and Amazon Simple Queue Service (Amazon SQS)

  • Amazon EventBridge: A serverless event bus service that connects applications using data from a variety of sources.
  • Amazon Simple Notification Service (Amazon SNS): A managed service for pub/sub messaging and mobile notifications, facilitating communication between decoupled components of applications.
  • Amazon Simple Queue Service (Amazon SQS): A message queuing service that enables decoupling and scaling of microservices, distributed systems, and serverless applications.

Business application services of Amazon Connect and Amazon Simple Email Service (Amazon SES)

  • Amazon Connect: A cloud-based contact center service that provides a scalable and customizable solution for customer service, with easy setup and pay-as-you-go pricing.
  • Amazon Simple Email Service (Amazon SES): A cost-effective, flexible, and scalable email service that enables developers to send mail from within any application.

Customer engagement services of AWS Activate for Startups, AWS IQ, AWS Managed Services (AMS), and AWS Support

  • AWS Activate for Startups: Provides startups with the resources needed to get started on AWS, including credits, training, and support.
  • AWS IQ: Connects customers with AWS-certified freelancers and firms for on-demand project work.
  • AWS Managed Services (AMS): Offers complete operations management for AWS, including infrastructure provisioning, monitoring, and maintenance.
  • AWS Support: Provides a range of plans for technical support, operational guidance, and architectural best practices, aiding businesses in their AWS journey.

Developer tool services and capabilities

  • AWS AppConfig: Helps manage, deploy, and scale application configurations quickly and safely.
  • AWS Cloud9: A cloud-based integrated development environment (IDE) for writing, running, and debugging code.
  • AWS CloudShell: Provides command-line access to AWS resources directly from the AWS Management Console.
  • AWS CodeArtifact: A fully managed artifact repository service for sharing and managing software packages.
  • AWS CodeBuild: A fully managed continuous integration service that compiles source code, runs tests, and produces software packages.
  • AWS CodeCommit: A managed source control service hosting private Git repositories.
  • AWS CodeDeploy: Automates software deployments to various AWS services.
  • AWS CodePipeline: A continuous integration and continuous delivery service for fast and reliable application updates.
  • AWS CodeStar: Provides a unified user interface for managing software development activities.
  • AWS X-Ray: Helps developers analyze and debug distributed applications in production.

End-user computing services

  • Amazon AppStream 2.0: A fully managed application streaming service for delivering desktop applications to any device.
  • Amazon WorkSpaces: A managed, secure Desktop-as-a-Service (DaaS) solution.
  • Amazon WorkSpaces Web: A low-cost, fully managed WorkSpace for accessing web applications.

Frontend web and mobile services

  • AWS Amplify: A set of tools and services for building secure, scalable mobile and web applications.
  • AWS AppSync: A managed service using GraphQL to develop applications with real-time and offline capabilities.

IoT services

  • AWS IoT Core: Allows connected devices to easily and securely interact with cloud applications and other devices.
  • AWS IoT Greengrass: An IoT open-source edge runtime and cloud service for building, deploying, and managing device software.

Compute purchasing options

  • On-Demand Instances: Pay for compute capacity by the hour or second with no long-term commitments.
  • Reserved Instances: Provides significant discount compared to On-Demand pricing, in exchange for a commitment to use a specific instance type.
  • Spot Instances: Allows you to bid on unused AWS capacity at a lower price.
  • Savings Plans: Flexible pricing models that offer lower rates compared to On-Demand pricing in exchange for a commitment to a consistent amount of usage.
  • Dedicated Hosts: Physical servers with EC2 instance capacity fully dedicated to your use.
  • Dedicated Instances: Instances that run in a VPC on hardware that’s dedicated to a single customer.
  • Capacity Reservations: Reserve capacity for your EC2 instances in a specific Availability Zone for any duration.

Data transfer charges

Data transfer charges in AWS are fees associated with the transfer of data over the network. This includes data transfer into and out of AWS services like Amazon S3 and EC2. Data transfer in between AWS services within the same region is generally free, but transferring data between regions or out to the internet incurs charges. These charges vary based on the amount of data transferred and the regions involved.

Storage options and tiers

AWS offers various storage services with different tiers based on access frequency, performance, and cost:

  • Amazon S3: Offers tiers like Standard for frequent access, Standard-IA (Infrequent Access) for less frequent access, and Glacier for archival storage.
  • Amazon EBS: Provides block storage with different volume types like General Purpose (SSD), Provisioned IOPS (SSD), and Magnetic for different performance needs.
  • Amazon EFS: File storage with standard and infrequent access tiers.

Billing support and information

AWS provides billing support through the AWS Management Console, which includes detailed billing reports and the AWS Cost Explorer for analyzing and managing AWS costs and usage. AWS also offers a Billing Dashboard, budgeting tools, and personalized alerts. AWS Support plans, available at various tiers (Basic, Developer, Business, and Enterprise), offer additional billing support.

Pricing information for AWS services

Pricing information for AWS services is available on the individual service pages on the AWS website. AWS pricing is generally based on a pay-as-you-go model, with prices varying by service, usage, and specific features or options selected. AWS also offers a Pricing Calculator to estimate costs for specific configurations and services.

AWS Organizations

AWS Organizations is a service for managing and governing your environment as you grow and scale your AWS resources. It allows you to centrally manage billing; control access, compliance, and security; and share resources across your AWS accounts. It supports organizing your accounts into groups and automating AWS services through policies.

AWS cost allocation tags

AWS cost allocation tags are labels that you can attach to AWS resources to organize and track your AWS costs. They allow you to break down your AWS bill and understand how costs are accrued by different teams, projects, or environments. There are two types of tags: AWS-generated tags created by AWS services and user-defined tags that you create and manage.

Resources and documentation available on official AWS websites

Official AWS websites provide a wealth of resources and documentation, including:

  • AWS Documentation: Comprehensive guides for all AWS services.
  • Whitepapers and Guides: In-depth resources on cloud computing, best practices, and architectural guidance.
  • AWS Training and Certification: Information on training courses and certifications for different roles and technical levels.
  • AWS Blogs and Articles: Insights and updates from AWS experts and the community.
  • AWS Forums and Discussion Boards: Platforms for asking questions and sharing knowledge with the AWS community.

AWS Support plans

AWS offers several support plans:

  • Basic: Access to documentation, whitepapers, and support forums. Basic support for billing and account questions.
  • Developer: Business hours email access to Cloud Support Associates for development and testing.
  • Business: 24/7 phone, email, and chat access to Support Engineers. Guidance for AWS best practices.
  • Enterprise: All Business plan benefits plus a Technical Account Manager (TAM) and Infrastructure Event Management.

Role of the AWS Partner Network, including independent software vendors and system integrators

The AWS Partner Network (APN) includes independent software vendors (ISVs) and system integrators (SIs) that help customers design, architect, build, migrate, and manage cloud solutions built on AWS. ISVs provide software solutions hosted on or integrated with the AWS platform, while SIs offer professional services to help clients deploy and manage AWS applications and services.

AWS Support Center

The AWS Support Center is the hub for accessing AWS Support. Users can open and track the status of support cases, communicate with AWS Support teams, and access a range of support resources. It’s a centralized place to get assistance and technical guidance, manage contacts, and review AWS Trusted Advisor recommendations for optimizing AWS environments.
