Configuring OpenSSH on Windows for AWS EC2

Step 1: Create a Key Pair in AWS Console

  1. Log in to the AWS Management Console and navigate to the EC2 dashboard.
  2. In the navigation pane, under “Network & Security”, click on Key Pairs.
  3. Click on Create Key Pair.
  4. Give your key pair a name (e.g., my-aws-key), select “pem” as the file format, and click Create.
  5. The private key file (my-aws-key.pem) will automatically be downloaded. Save this file; it’s used to securely access your EC2 instances.

Step 2: Save the Key on Windows

  1. Create a Directory for SSH Keys:
    • It’s a good practice to store all your SSH keys in one place. You can create a folder named .ssh in your user directory (e.g., C:\Users\YourUsername\.ssh\).
    • Move the downloaded .pem file to this folder.

Step 3: Configure OpenSSH

  1. Install OpenSSH (if not already installed):

    • Windows 10 and newer versions have OpenSSH client installed by default. You can verify its installation by opening the Command Prompt or PowerShell and running ssh -V.

  2. Set Permissions for the Key File:

    • Right-click on the .pem file, select “Properties”, then go to the “Security” tab.
    • Click “Edit” and set permissions so that only your user account has read access.

  3. Create an SSH Config File:

    • In the same .ssh directory, create a file named config (no file extension).
    • Open this file with a text editor (like Notepad).

  4. Edit the Config File:

    • Add the following lines to the config file:

      Host *
    IdentityFile C:\Users\YourUsername\.ssh\my-aws-key.pem

    • Replace YourUsername with your Windows username and my-aws-key.pem with the name of your private key file.

    • This configuration tells SSH to use this key for all hosts you connect to.

  5. Connecting to Your EC2 Instance:

    • To connect to an instance, open Windows Terminal or Command Prompt and use the command:

      ssh ec2-user@your-instance-ip

    • Replace ec2-user with the appropriate username for your AMI (e.g., ubuntu for Ubuntu AMIs) and your-instance-ip with the public IP address of your EC2 instance.

Important Security Notes

  • Keep your private key secure and do not share it.
  • Regularly check and update your SSH configurations for any changes or security enhancements.
