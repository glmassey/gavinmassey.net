Netstat Quick Reference

Introduction

Netstat, a command-line tool available in Windows Server, is a powerful utility for network diagnostics and statistics. It provides detailed information about network connections, routing tables, interface statistics, and more. This tool is essential for network administrators and IT professionals to monitor, diagnose, and troubleshoot network-related issues on a Windows Server. Below is a list of common usages of Netstat on Windows Server.

List of Netstat Usages on Windows Server

  1. Display All Active Connections:

    netstat -a

    Lists all active TCP and UDP connections and listening ports.

  2. Show Listening Ports:

    netstat -an | findstr "LISTENING"

    Lists all ports on which the server is listening, filtering to show only those in a LISTENING state.

  3. Identify Open Ports:

    netstat -an

    Shows all ports that are currently open, including listening, established, and other states.

  4. Show Network Statistics for Each Protocol:

    netstat -s

    Displays detailed statistics by protocol, including TCP, UDP, ICMP, and IP.

  5. Display Routing Table Information:

    netstat -r

    Shows the routing table, equivalent to the ‘route print’ command.

  6. Show Interface Statistics:

    netstat -e

    Provides statistics for each network interface, including bytes, unicast packets, non-unicast packets, discards, errors, and unknown protocols.

  7. Check for Listening Services or Daemons:

    netstat -ab

    Lists ports in listening state along with the executable involved. This is helpful for identifying which application is listening on a particular port.

  8. Find Programs Using Network Sockets:

    netstat -o

    Shows active connections and listening ports with the process ID (PID) of each process.

  9. Filter Network Information for a Specific Protocol (e.g., TCP):

    netstat -at

    Filters the output to show only TCP protocol connections and listening ports.

  10. Troubleshoot Network Issues (e.g., Show Statistics Every 5 Seconds):

    netstat -e 5

    Refreshes and displays Ethernet statistics every 5 seconds. Useful for monitoring network activity in real time.

These commands should be run in a Command Prompt or PowerShell window, and some might require elevated privileges (Run as Administrator) to display complete information. The examples are tailored for use on Windows Server environments for various network monitoring and troubleshooting tasks.
