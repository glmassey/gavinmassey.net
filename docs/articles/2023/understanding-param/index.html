Understanding the `param` Keyword in PowerShell

In PowerShell, the param keyword is used to define parameters that a script or function can accept. Parameters allow you to pass values into scripts and functions, making them more flexible and reusable. By providing different arguments, users can tailor the behavior of a script or function without modifying its actual code.

Features of the param Block:

  1. Positional Parameters: By default, parameters are positional, and values are assigned based on the order they appear in the param block.

  2. Mandatory Parameters: Using the [Parameter(Mandatory=$true)] attribute, you can require the user to provide a value for a specific parameter.

  3. Default Values: Parameters can have default values specified with the = operator.

  4. Type Constraints: You can specify the data type of a parameter to ensure that only values of that type are accepted.

  5. Validation: With attributes like [ValidateRange()], you can add validation to parameters.

  6. Parameter Sets: You can create multiple sets of parameters that a function can accept, allowing for different combinations of arguments.

  7. Pipeline Input: Parameters can be designed to accept input directly from the pipeline.

Basic Syntax:

param (
    [type]$ParameterName = DefaultValue,
    [type]$AnotherParameter
)

Examples:

  1. Basic Parameter:
param (
    $Name
)
Write-Output "Hello, $Name!"
  1. Mandatory Parameter:
param (
    [Parameter(Mandatory=$true)]
    $Name
)
Write-Output "Hello, $Name!"
  1. Typed Parameter with Default Value:
param (
    [string]$Greeting = "Hello"
)
Write-Output "$Greeting, World!"
  1. Parameter Accepting Pipeline Input:
param (
    [Parameter(ValueFromPipeline=$true)]
    $Name
)
Process {
    Write-Output "Hello, $Name!"
}

Benefits:

  • Flexibility: Parameters allow users to provide different inputs to modify script behavior.
  • Reusability: With parameters, the same script or function can handle a variety of scenarios.
  • Readability: Named parameters make scripts and functions more self-documenting, aiding in understanding the expected inputs.
