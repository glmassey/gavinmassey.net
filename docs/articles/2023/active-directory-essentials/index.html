Active Directory Essentials: Groups, Powershell, and Delegation

Simplifying Active Directory Group Scopes

Understanding Group Types

  1. Domain Local Groups: Restricted to a single domain. They manage access within their domain, but not in other domains.

  2. Global Groups: Span across all domains in a forest. Useful for organizing users within a domain and granting them resource access across the forest.

  3. Universal Groups: Operate across the entire Active Directory forest. Ideal for large-scale environments needing access across multiple domains.

Security vs. Distribution Groups

  • Security Groups: Used for assigning permissions to resources. They can be Domain Local, Global, or Universal.

  • Distribution Groups: Mainly for email distribution; they don’t control resource access.

Choosing the Right Group

  • Domain Local Groups: Best for managing resources within one domain.
  • Global Groups: Good for organizing users in one domain and accessing resources in other domains within the forest.
  • Universal Groups: Best for complex, multi-domain environments for broad access.

Rejoining a Computer to the Domain with PowerShell

To rejoin a broken trust relationship between a domain and a computer:

  1. Open PowerShell as Administrator: Right-click on Start and choose “Windows PowerShell (Admin)”.

  2. Repair Command: Type Test-ComputerSecureChannel -Repair and hit Enter. Restart the PC if the command returns True.

  3. Alternative Method: If the command fails, use Remove-Computer and Add-Computer cmdlets for a full rejoin.

Delegating Control: Password Reset Example

Delegate Control to a User

  1. Access Active Directory Users and Computers: Press Windows Key + R, type dsa.msc, and press Enter.

  2. Right-Click on the OU: Select “Delegate Control”.

  3. Use the Delegation Wizard: Follow the steps to add a user and set permissions for password resets.

Creating a Custom MMC for Delegated User

  1. Open MMC: Press Windows Key + R, type mmc, and press Enter.

  2. Add Snap-in: Choose “Active Directory Users and Computers”, focusing on the specific OU.

  3. Save the Customized MMC: Save and distribute to the delegated user.

Important Notes

  • Ensure the user has the necessary permissions to use the MMC.
  • Admin privileges are required for these tasks.
