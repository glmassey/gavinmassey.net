PowerShell Quickstart Guide

PowerShell Quickstart Guide

1. Introduction

PowerShell is Microsoft’s task automation framework, combining command-line speed, the flexibility of scripting, and the power of a GUI-based admin tool. Whether you’re automating tasks, extracting and processing data, or managing systems, PowerShell is a versatile tool to know.

2. Launching PowerShell

You can find PowerShell by searching for it in the Windows start menu. For administrative tasks, it’s advisable to run PowerShell as an administrator (right-click the application and select “Run as administrator”).

3. The Basics

When you launch PowerShell, you’re presented with a command prompt where you can start typing commands, known as cmdlets (pronounced “command-lets”).

Cmdlets use a Verb-Noun format, describing the action and the target, respectively. Examples include:

  • Get-Help
  • Start-Service
  • New-Item

4. Exploring Cmdlets

To list all available cmdlets:

Get-Command

To find cmdlets related to a specific noun (e.g., Service):

Get-Command -Noun Service

5. Getting Help

Before diving deep, familiarize yourself with the Get-Help cmdlet. It’s a lifeline!

For detailed information on a cmdlet:

Get-Help Set-ExecutionPolicy -Detailed

For examples of how to use a cmdlet:

Get-Help New-Item -Examples

6. Basic Commands

Navigating Directories: Much like the old DOS command prompt or UNIX shell:

cd C:\Users

Listing Items in a Directory:

Get-ChildItem   # Equivalent to 'dir' in command prompt

Creating a New Folder:

New-Item -Path "C:\Users\YourName" -Name "NewFolder" -ItemType Directory

7. Working with Processes and Services

View Running Processes:

Get-Process

Start and Stop a Service:

Start-Service -Name "wuauserv"
Stop-Service -Name "wuauserv"

8. Scripting

For automation, you can write a series of cmdlets in a .ps1 file.

Example: A script to list all processes consuming more than 100MB of RAM:

$highMemProcesses = Get-Process | Where-Object { $_.WS -gt 100MB }
$highMemProcesses | Format-Table Name, WS -AutoSize

To run a script:

.\path\to\script.ps1

Important: By default, Windows may not allow script execution for security reasons. Adjust this with Set-ExecutionPolicy (consult its help before changing).

9. Beyond the Basics

This guide is just the tip of the iceberg. PowerShell’s real power is unlocked through its object-oriented nature, pipelines, advanced scripting capabilities, and integration with .NET.

10. Final Tips

  • Tab Completion: Start typing a cmdlet, then press Tab to cycle through cmdlets that match.

  • Pipelines: Use the | character to send the output of one cmdlet as input to another, chaining operations.

  • Stay Updated: Microsoft continually evolves PowerShell, with new cmdlets and features. Regularly check for updates.
