Lab 01: Terraform Hello World with AWS

Overview

In this lab, we’ll use Terraform to deploy the latest Ubuntu LTS server on AWS within a custom VPC. We’ll be using an AWS free tier eligible instance type to ensure cost-effectiveness while learning.

Objectives

  • Set up a basic Terraform configuration with separate files for different resources.
  • Create a custom VPC and deploy an Ubuntu LTS server on AWS within this VPC.
  • Understand the basics of Terraform syntax and AWS resources, including VPCs, EC2 instances, and security groups.

Prerequisites

  • AWS account with free tier eligibility.
  • Basic knowledge of AWS services, especially EC2 and VPC.
  • Terraform installed on your machine. Download Terraform
  • AWS CLI installed and configured. AWS CLI Installation

Lab Setup

  1. Initialize Your Workspace:

    • Clone this repository to your local machine.
    • Navigate to the lab-01-terraform-hello-world directory.

  2. AWS Credentials:

    • Ensure your AWS credentials are configured. This typically involves setting up the AWS CLI with aws configure.
    • Security Note: Never hardcode your AWS credentials in your Terraform files.

  3. Terraform Files:

    • You will be working with several files:
      • vpc.tf: Contains the Terraform configuration for creating a custom VPC.
      • ec2.tf: Contains the Terraform configuration for deploying an EC2 instance within the new VPC.
      • outputs.tf: Defines the output variables like the public IP address of the EC2 instance.
      • variables.tf: Contains the variable definitions used in other Terraform files.

  4. Terraform Initialization:

    • Run terraform init in your project directory. This command initializes Terraform and downloads the necessary providers.

Deployment Steps

  1. Review the Configuration:

    • Examine the contents of vpc.tf, ec2.tf, variables.tf, and outputs.tf. Understand the resources being created and their configurations.

  2. Plan Your Deployment:

    • Run terraform plan to preview the changes Terraform will make.

  3. Apply the Configuration:

    • Execute terraform apply to create the resources. Confirm the action when prompted.
    • Once applied, Terraform will output the public IP address of the newly created Ubuntu server.

  4. Access the Server:

    • Connect to the server via SSH. You will need the private key corresponding to the key pair used in the EC2 instance setup.

  5. Clean Up Resources:

    • After you are done, make sure to destroy the resources to avoid any charges. Run terraform destroy.

Notes

  • Be mindful of AWS usage to avoid charges. Regularly check your AWS Console to monitor running services.
  • Always follow best practices for security, such as minimal privilege IAM roles and secure key management.

Installing Terraform

  1. Download the binary from Hashicorp.

  2. Extract and place in a suitable location.

  3. Add location to system path, the following command may be used depending on where you put it. You will need to execute the following command from an administrative PowerShell session.

    $path = [System.Environment]::GetEnvironmentVariable("Path", [System.EnvironmentVariableTarget]::Machine)
$newPath = "$path;C:\Program Files\HashiCorp\Terraform\terraform_1.6.5_windows_amd64"
[System.Environment]::SetEnvironmentVariable("Path", $newPath, [System.EnvironmentVariableTarget]::Machine)
