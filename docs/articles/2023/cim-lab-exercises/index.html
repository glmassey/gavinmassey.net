WIM and CIM Lab Exercises

Lab Exercise 1: System Overview

Objective: Get an overview of your computer system’s properties.

  1. Task: Retrieve information about the operating system, including its version, last boot up time, and the number of users currently logged on.

    • Command Hint: Get-CimInstance -ClassName CIM_OperatingSystem

  2. Task: Fetch the computer’s name, manufacturer, model, and the total physical memory.

    • Command Hint: Get-CimInstance -ClassName Win32_ComputerSystem

Lab Exercise 2: Disk Analysis

Objective: Analyze the disk space usage and health.

  1. Task: List all logical disks, including their size, free space, and filesystem type.

    • Command Hint: Get-CimInstance -ClassName Win32_LogicalDisk

  2. Task: Fetch the details of all partitions on your main hard drive, including partition type and size.

    • Command Hint: Get-CimInstance -ClassName Win32_DiskPartition

Lab Exercise 3: Process and Services

Objective: Understand the processes and services running on your system.

  1. Task: Display all currently running processes on your machine.

    • Command Hint: Get-CimInstance -ClassName Win32_Process

  2. Task: Retrieve a list of all services on your machine, including their status (running, stopped, etc.).

    • Command Hint: Get-CimInstance -ClassName Win32_Service

  3. Task: Try starting or stopping a specific service. (Be cautious and select a non-critical service.)

    • Command Hint: Invoke-CimMethod -ClassName Win32_Service -MethodName 'StartService' -Filter "Name='YourServiceName'"

Lab Exercise 4: Network Configuration

Objective: Explore the network configuration and status of your computer.

  1. Task: List all the network adapters on your system.

    • Command Hint: Get-CimInstance -ClassName Win32_NetworkAdapter

  2. Task: Fetch details about the IP configuration of your primary network adapter.

    • Command Hint: Get-CimInstance -ClassName Win32_NetworkAdapterConfiguration | Where-Object { $_.IPEnabled -eq $true }

Lab Exercise 5: System Health and Configuration

Objective: Examine the hardware components and system configurations.

  1. Task: Display details about the system’s BIOS.

    • Command Hint: Get-CimInstance -ClassName Win32_BIOS

  2. Task: Retrieve detailed information about physical memory (RAM) modules.

    • Command Hint: Get-CimInstance -ClassName Win32_PhysicalMemory

  3. Task: Fetch the system’s current power settings and configurations.

    • Command Hint: Get-CimInstance -ClassName Win32_PowerPlan

Lab Exercise 6: System Overview and Data Export

Objective: Obtain a system overview and export the data for further analysis.

  1. Task: Retrieve the computer’s name, manufacturer, model, and total physical memory, then save the output to a CSV file.
    • Command Hint: Get-CimInstance -ClassName Win32_ComputerSystem | Export-Csv -Path "C:\path_to_your_directory\system_overview.csv"

Lab Exercise 7: Disk Analysis and Space Management

Objective: Analyze disk space and free up space by cleaning temporary files.

  1. Task: List all logical disks, their size, free space, and filesystem type, and then save this data to a CSV file.

    • Command Hint: Get-CimInstance -ClassName Win32_LogicalDisk | Export-Csv -Path "C:\path_to_your_directory\disk_overview.csv"

  2. Task: If your C: drive has less than 10GB free, attempt to clean the temp directory.

    • Commands:
      1. Check space: $space = (Get-CimInstance -ClassName Win32_LogicalDisk | Where-Object { $_.DeviceID -eq 'C:' }).FreeSpace
      2. Clean temp if necessary: if ($space -lt 10GB) { Remove-Item "C:\Windows\Temp\*" -Recurse -Force }

Lab Exercise 8: Remote Process Management

Objective: Remotely monitor and control processes on another machine (ensure PS remoting is enabled on the remote machine).

  1. Task: Display the top 5 memory-consuming processes from a remote machine named ‘RemotePC’.

    • Command Hint: Get-CimInstance -ClassName Win32_Process -ComputerName "RemotePC" | Sort-Object WS -Descending | Select-Object -First 5

  2. Task: Remotely stop a specific non-critical process (e.g., notepad.exe) on ‘RemotePC’.

    • Command Hint: Get-CimInstance -ClassName Win32_Process -ComputerName "RemotePC" | Where-Object { $_.Name -eq 'notepad.exe' } | Invoke-CimMethod -MethodName "Terminate"

Lab Exercise 9: Remote Network Configuration Inspection

Objective: Examine the network configuration of a remote computer.

  1. Task: Fetch details about the IP configuration of the primary network adapter of a computer named ‘RemotePC’ and save this to a CSV.
    • Command Hint: Get-CimInstance -ClassName Win32_NetworkAdapterConfiguration -ComputerName "RemotePC" | Where-Object { $_.IPEnabled -eq $true } | Export-Csv -Path "C:\path_to_your_directory\remote_network_config.csv"

Lab Exercise 10: Remote System Health Checks and Action

Objective: Monitor hardware health remotely and take action based on findings.

  1. Task: Check the status of a remote system’s battery. If its status is not “OK”, send an email alert.
    • Commands:
      1. Check battery: $batteryStatus = (Get-CimInstance -ClassName Win32_Battery -ComputerName "RemotePC").Status
      2. Send email if necessary: if ($batteryStatus -ne "OK") { Send-MailMessage -From 'your_email@example.com' -To 'admin_email@example.com' -Subject 'Battery Alert' -Body 'The remote PC battery is not OK.' -SmtpServer 'your_smtp_server' }
